From the solution explorer, select `designAutomationsample` project, under the `Properties`,
select and open the `launchSettings.json`

- Change the `applicationUrl` to "http://localhost:8080"
- Go to profiles\forgeSample\environmentVariables add following environments.

```bash
ASPNETCORE_URLS: use http://localhost:8080
APS_CLIENT_ID: use your id here
APS_CLIENT_SECRET: use your secret here
APS_WEBHOOK_URL: use the ngrok forwarding URL from previous step
```

- Select `designAutomationSample` project, right-click `Add`->`New Item`, select `JSON`.
- Name it as `appsettings.user.json`.
- Add following configuration settings, this is required by [DesignAutomation SDK](https://www.nuget.org/packages/Autodesk.Forge.DesignAutomation)
  to create oAuth and run API requests.

```json title=appsettings.user.json
{
  "Forge": {
    "ClientId": "your client id",
    "ClientSecret": "your secret"
  }
}
```

As this is running locally, `sslPort` is set to 0. It should look like as shown below.

```json title="launchSettings.json"
{
  "iisSettings": {
    "windowsAuthentication": false,
    "anonymousAuthentication": true,
    "iisExpress": {
      "applicationUrl": "http://localhost:8080",
      "sslPort": 0
    }
  },
  "profiles": {
    "forgeSample": {
      "commandName": "Project",
      "dotnetRunMessages": true,
      "launchBrowser": true,
      "applicationUrl": "http://localhost:8080",
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development",
        "APS_CLIENT_ID": "your client id here",
        "APS_CLIENt_SECRET": "your secret here",
        "APS_WEBHOOK_URL": "your ngrok url here"
      }
    },
    "IIS Express": {
      "commandName": "IISExpress",
      "launchBrowser": true,
      "environmentVariables": {
        "ASPNETCORE_ENVIRONMENT": "Development"
      }
    }
  }
}
```
